You do not need to randomly select <%= Basics_Users %> at the individual level. It may be more feasible to select by classroom or by school. However, your ability to learn whether <%= Basics_Tech_Name %> is effective will be stronger if you can choose at the individual level.

Creating your dataset

After you have identified the data elements and sources, the next step is to combine all data elements into one tidy dataset. We recommend generating tidy datasets not only because doing so is a requirement to use the RCE Coach and most statistical software packages, but also because a tidy dataset is easy to manipulate, model, and visualize.

If you do not have access to statistical software, you can use Microsoft Excel to prepare your data. An Excel template is available to help you organize your data.

Uploading a data set that does not follow the guidelines might lead to problems in running your analysis. Please read our Prepare Your Data for Randomization guide for more information.

When you finish this step, your data set should look similar to the one below.

Each row represents a single observation. For example, the current highlighted row is for the student with ID 834305.

Each variable (e.g. anonymous student ID, grade, fall score) has its own column.

Valid values for categorical variables are 1 (yes or true) and 0 (no or false).

Missing data is replaced with NA (not available) so that the Coach skips that cell.

You will have to merge your existing datasets into one complete dataset. Your dataset should have one row for each of your <%= Basics_Users %> and one column for each data field or variable. This will be easy to do using the unique identifiers.

If you use Excel to manage your data you can do this using a VLOOKUP function.

<% var checked = ( eval.prepareRandom.Check_Numeric == "Numeric_Checked" ) ? "checked" : ""; %> />

You will have to convert all categorical variables to numerical dummy variables with values of 1 for yes or true and 0 for no or false. For example, if your background characteristics include gender as a variable, you might have M or F in each cell of that column. Instead, you should change the variable from Gender to Female, and change each cell that indicates the participant is a female to 1, and each cell that indicates the participant is male to 0.

If your categorical variable contains more than two options, such as Race—where the options are Asian, Black, White, and Other—you will have to create a binary or dummy variable for each option. For example, Asian would be one column (with 0 representing non-Asians and 1 representing Asians), Black would be a second column, White would be a third column, and Other would be a fourth column.

If you have merged datasets and there are missing data, make sure you are consistently coding missing values as NA. You want to be extra careful that missing data does not have a number value, such as 0 or 999. These values will get incorporated into the analysis.

Checking the quality of your data

After constructing your data file and converting your variables, the final step is to check the quality of your data. You can run the following checks to identify potential data issues that warrant additional investigation:

This check helps to identify extremely low or high values that are outliers in your distribution or can signal a special missing code that has to be converted to NA. You might want to check with someone who is familiar with the data to confirm the value range makes sense.

If you are working in Excel you can use MIN and MAX functions to easily find these values.

The tools, and some statistical software packages, will automatically drop observations missing key data. You should try to understand why data are missing and how excluding <%= Basics_Users %> with incomplete data will affect your results.

If you are working in Excel you can sort and filter your data to view missing values. To determine exactly how many values are missing for a single variable you can use the COUNTIF function; to determine how many observations have at least one missing value you can use a nested COUNTIF with OR function.
