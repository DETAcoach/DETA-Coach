Creating your data set

Once you have identified the data elements and data sources, the next step is to combine all data elements into one tidy dataset. We recommend generating tidy datasets not only because it is a requirement to use the RCE Coach and most statistical software packages, but also because a tidy dataset is easy to manipulate, model, and visualize.

If you do not have access to statistical software, you can use Microsoft Excel to prepare your data. An Excel Template is available to help you organize your data.

Uploading a data set that does not follow the guidelines outlined below may lead to problems in running your analysis. Please read our Prepare Your Data Set for Analysis guide for more information

When you are finished with this step, your data set should look similar to the one below.

Each row represents a single observations (e.g. treatment, spring score, school id). For example, the current highlighted row is for the student with id 834305.

Each variable has its own column.

Valid values for categorical variables are 1 (yes or true) and 0 (no or false).

Missing data is replaced with "NA" so that the Coach skips that cell.

You will need to merge your existing datasets into one complete dataset. Your data set should have one row for each one of your <%= Basics_Users %> and one column for each data field or variable. This will be easy to do using the unique identifiers.

If you are using Excel to manage your data you can do this using a VLOOKUP function.

<% var checked = ( eval.prepare.Check_Numeric == "Numeric_Checked" ) ? "checked" : ""; %> />

You will need to convert all categorical variable to numerical dummy variables with values of 1 for yes or true and 0 for no or false. For example, if your background characteristics include gender as a variable, you may have “M” or “F” in each cell of that column. Instead, you should change the variable from ‘Gender’ to ‘Female’, and change each cell that indicates the participant is a female to 1, and each cell that indicates the participant is male to 0.

If your categorical variable contains more than two options, such as “Free or Reduced Price Lunch” (where the options are (1) free, (2) reduced, and (3) full price) you will need to create a dummy variable for each option. For example, Free would be one column (with 0 representing that they don’t receive a free lunch and 1 representing that they do), Reduced price would be a second column, and Paid (meaning they pay full price) would be a third column.

<% var checked = ( eval.prepare.Check_Missing == "Missing_Checked" ) ? "checked" : ""; %> />

If you have merged datasets and there is missing data, make sure you are consistently coding that as “NA” to ensure that the Coach can analyze your data. You want to be extra careful to make sure missing data has not been given a number designation, like 0 or 999. These values will get incorporated into the analysis.

Checking the quality of your data

After constructing your data file and converting your variables, the final step is to check the quality of your data. The following checks can be run to identify potential data issues that warrant additional investigation:

<% var checked = ( eval.prepare.Check_Min_Max == "Min_Max_Checked" ) ? "checked" : ""; %> />

This check may help to identify extremely low or high values that are outliers in your distribution or may signal a special missing code that needs to be converted to a missing value. You may want to check with someone who is familiar with the data to confirm the value range makes sense.

If you are working in Excel you can use MIN and MAX functions to easily find these values.

<% var checked = ( eval.prepare.Check_Miss_Impact == "Miss_Impact_Checked" ) ? "checked" : ""; %> />

The tools, and some statistical software packages, will automatically drop observations that contain missing data. You should try to understand why data is missing and how excluding <%= Basics_Users %> with incomplete data will affect your results.

If you are working in Excel you can sort and filter your data to view missing values. To determine exactly how many values are missing for a single variable you can use the COUNTIF function, and to determine how many observations have at least one missing value you can use a nested COUNTIF with OR function.
