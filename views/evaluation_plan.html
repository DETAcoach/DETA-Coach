<% if(eval.path === "path-random") { %>

This evaluation will use a randomized pilot. The choice of who will pilot the technology will be determined by chance, like a coin flip. This design is the best way to determine if the educational technology is moving the needle because it creates two groups that are similar on both observed characteristics (such as prior achievement) and unobserved characteristics (such as motivation). This means that we can be confident that any differences we see in outcomes are due to the technology and not other factors.

will be randomly selected to use the technology.
  • Why does this matter? There are multiple ways to randomly assign individuals: we could randomly select individuals to use the technology, or we could randomly select groups of individuals (such as schools) to use the technology. In general, the lower the level of randomization, the stronger ones ability to make conclusions, but it is not always feasible to randomly assign at the individual level.

<% } %> <% if(eval.path === "path-matching") { %>

This evaluation will use a matched comparison design. We will create a comparison group of non-users similar to <%= Basics_Users %> who already using the technology. This design will allow us to determine if the educational technology is moving the needle because it allows us to create two groups that are similar on observed characteristics (such as prior achievement). By comparing similar groups, we can be more confident that differences we see in outcomes are due to the technology and not other observed factors. However, matching does not allow us to create groups that are similar on unobserved characteristics, so it is possible that differences could be attributed to unobserved factors.

Things to Consider

It can be helpful to think backwards from the date you plan to implement the technology or the date you want to complete your evaluation.

Even if you don’t know exact dates it will be helpful for planning purposes to estimate when these steps will occur.

If you do not know who will be responsible for each milestone, you can leave it as “To be determined.” The meeting with other stakeholders can be used to identify who will be responsible for different parts of the evaluation.

It can be helpful to check how the technology is actually being used at some point (or multiple times) during the evaluation. If the technology is not being used as expected, you will want to know this sooner rather than later in order to make any necessary adjustments to the implementation. For example, if you are piloting a personalized reading technology with students you may want to look at student usage data a month after implementing the technology to see if students have been using it. If it looks like students are not using the technology, there may be a problem with the technology implementation. To learn more about what this requires see [LINK TO CHECK USAGE DOCUMENT].
